# browser-clustering

ブラウザ上で完結する（サーバー送信なし）`k-means`クラスタリングWebアプリの要件定義です。  
実装は `TypeScript/JavaScript` 前提で、`pnpm + Node.js + Vite + Biome + Vitest + shadcn/ui` を基本技術スタックとします。

## 1. 目的

- ユーザーがアップロードした `CSV` または `TSV` をブラウザ内だけで解析し、`k-means` でクラスタリングする。
- オプションでクラスタリング前に `PCA` 次元圧縮を行う。
- クラスタリング結果を `PCA 2D` または `PCA 3D` で可視化する。
- 可視化結果は hover で情報表示し、クラスタ名はユーザーが後から編集できる。
- 結果を `画像(PNG/SVG)`、`HTML(インタラクティブ)`、`CSV(カテゴリラベル付き)` でエクスポートする。

## 2. スコープ

### 対象

- ブラウザ実行のみのクライアントアプリ
- 数値特徴量を使った `k-means` クラスタリング
- `PCA` による前処理・可視化
- ローカルエクスポート

### 非対象

- サーバー処理・クラウド保存・外部API送信
- 分散学習やGPUサーバー利用
- 自動特徴量エンジニアリング（カテゴリ変数自動エンコード等）

## 3. 入力仕様

### ファイル形式

- 受け付け: `CSV`, `TSV`
- 区切り文字: 自動判定（`,` または `\t`）

### 列構成

- 先頭1列: ID/名称ラベル（文字列）
- 2列目以降: 数値特徴量

### 欠損値・非数値セル

- 欠損値または非数値セルを含む行は除外する
- UIに除外件数と行番号を警告表示する

## 4. 前処理・クラスタリング仕様

### 標準化

- `z-score` 標準化をデフォルト `ON`
- ユーザーが `OFF` に切替可能

### クラスタ数 `k`

- ユーザー手動指定を基本とする
- 参考情報として `Elbow` と `Silhouette` による候補提示を行う
- 最終決定はユーザーが行う

### k-means実行

- 初期化: `k-means++`
- 反復戦略: `n_init > 1`（複数初期値）で最良解を採用
- 乱数シード: デフォルト値を持ち、ユーザーが手動変更可能

### クラスタリング前PCA（任意）

- オプションで `ON/OFF` 切替可能
- `ON` の場合は目標次元をユーザーが手入力

## 5. 可視化仕様

### 可視化次元

- クラスタリング後、可視化用に `PCA` を実行
- 可視化は `2D` / `3D` を選択可能

### 描画基盤

- `Plotly.js` を採用（2D/3D・hover・HTMLエクスポートを統一）

### hover表示

- 表示項目: `ID` + `cluster名` + `可視化座標`

### クラスタラベル編集

- `cluster_id` ごとにユーザーが表示名（`cluster_label`）を編集可能
- 編集結果はグラフ表示とCSV出力に反映

## 6. エクスポート仕様

### 画像

- `PNG`, `SVG`

### インタラクティブ

- `HTML`（ブラウザで単体表示できる形式）

### 表形式

- `CSV` 出力列: `元データ列 + cluster_id + cluster_label`

## 7. パフォーマンスとガードレール

- 初期推奨上限を `50万セル（行数 × 特徴量数）` とする
- 閾値超過時は警告と確認ダイアログを表示する
- ユーザー明示確認がある場合のみ処理を継続する
- READMEに「安全実行の目安」「重くなる条件」「中断推奨条件」を明記する

## 8. プライバシー・セキュリティ要件（外部送信ゼロ）

- CDN不使用（依存はビルド時に同梱）
- 外部ネットワークAPI呼び出しを禁止（`fetch/XHR/WebSocket` の外向き通信なし）
- 厳格な `CSP` を設定し、通信先を制限
- 本番ビルド時に外部参照の静的監査手順を用意

## 9. 対応ブラウザ

- 正式対象: 最新版 `Chrome/Edge`（Chromium系、最新2バージョン想定）

## 10. 画面要件（UI）

- UIコンポーネントは `shadcn/ui` を基本とする
- 最低限のアクセシビリティを確保する
- キーボード操作
- フォーム要素のラベル付与
- 警告・エラーの可視化

## 11. データ契約（実装インターフェース）

### 入力レコード（内部表現）

```ts
type InputRecord = {
  label: string; // 先頭列
  features: number[]; // 2列目以降
};
```

### 実行パラメータ

```ts
type RunConfig = {
  k: number;
  seed: number;
  useStandardize: boolean;
  usePcaBeforeClustering: boolean;
  pcaTargetDim?: number;
  vizDim: 2 | 3;
};
```

### 出力レコード（CSV向け）

```ts
type OutputRecord = {
  // 元データ列を保持
  [originalColumn: string]: string | number;
  cluster_id: number;
  cluster_label: string;
};
```

## 12. 受け入れ基準（テスト観点）

`Vitest` で少なくとも以下を自動テスト対象にする。

- パーサー
- CSV/TSV自動判定が正しい
- 欠損/非数値行の除外と警告件数が一致する
- 前処理・学習
- 標準化ON/OFFで挙動が切り替わる
- 同一seedで再現可能
- `k-means++ + n_init` が単回より不利にならない
- 候補提示
- Elbow/Silhouette候補が定義域内で安定して返る
- PCA
- 前処理PCAの次元数が指定通り
- 可視化PCAが2D/3Dで正しい座標形状を返す
- 可視化・ラベル
- hover項目（ID, cluster名, 座標）が表示される
- クラスタ名編集が表示・CSVに反映される
- エクスポート
- PNG/SVG/HTML/CSVが生成される
- CSV列構成が仕様通り
- ガードレール
- 50万セル超過時に確認ダイアログが出る
- オフライン保証
- 本番成果物に外部URL参照や外向き通信コードがない

## 13. 採用ロジックの根拠

- `z-score` 標準化、`k-means++` 初期化、`PCA`、`Silhouette/Elbow` は統計学・機械学習で広く確立された手法を採用
- 独自ヒューリスティックは最小化し、説明可能で再現可能な処理を優先
